# TPR Concession Method

Настольное приложение для программной реализации метода уступок в дискретной многокритериальной задаче принятия решений.

Проект выполнен для индивидуального задания по дисциплине "Теория принятия решений", вариант 22. Приложение позволяет вручную вводить альтернативы, критерии, направления оптимизации, порядок важности и уступки, затем строит пошаговый протокол решения и показывает итоговую рекомендуемую альтернативу или множество равноценных альтернатив.

## Технологии

- Python 3.11 или Python 3.12
- PySide6 для графического интерфейса
- pytest и pytest-cov для тестирования расчетного ядра
- PyInstaller для сборки exe-файла

## Метод уступок

Метод уступок упорядочивает критерии по важности. На каждом критерии, кроме последнего, находится оптимальное значение среди текущего множества альтернатив и применяется допустимая уступка.

Для max-критерия остаются альтернативы со значением:

```text
value >= optimum - concession
```

Для min-критерия остаются альтернативы со значением:

```text
value <= optimum + concession
```

Для процентной уступки абсолютная величина уступки считается как:

```text
abs(optimum) * concession / 100
```

На последнем критерии уступка не применяется: выбираются все альтернативы с лучшим значением последнего критерия.

## Установка

Для удобства проверки и демонстрации работы приложения готовая Release-версия программы уже собрана и находится в папке:

dist\TPR_Concession_Method.exe

Создать виртуальное окружение:

```powershell
py -3.12 -m venv .venv
```

Если Python 3.12 не установлен, можно использовать доступный Python 3.11 или 3.12:

```powershell
python -m venv .venv
```

Установить зависимости:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Запуск приложения

```powershell
.\.venv\Scripts\python.exe main.py
```

В окне приложения можно:

- задать количество альтернатив и критериев;
- создать таблицу исходных данных;
- вручную отредактировать значения;
- настроить направления `max`/`min`, порядок важности и уступки;
- загрузить пример 1, пример 2 или вариант 22;
- выполнить расчет и получить протокол для отчета.

## Запуск тестов

```powershell
.\.venv\Scripts\python.exe -m pytest
```

Короткий режим:

```powershell
.\.venv\Scripts\python.exe -m pytest -q
```

Проверка покрытия расчетного ядра:

```powershell
.\.venv\Scripts\python.exe -m pytest --cov=app.concessions --cov=app.models
```

Требуемое покрытие расчетного ядра по заданию: не ниже 80%.

## Сборка exe

Сборка выполняется через PyInstaller:

```powershell
build\build_release.bat
```

После успешной сборки exe-файл находится здесь:

```text
dist\TPR_Concession_Method.exe
```

Скрипт сборки использует `.venv\Scripts\python.exe`, если виртуальное окружение создано. Если `.venv` нет, используется системный `python`.

## Встроенные примеры

### Пример 1

Альтернативы:

| Альтернатива | K1 | K2 | K3 |
| --- | ---: | ---: | ---: |
| A1 | 100 | 20 | 8 |
| A2 | 95 | 10 | 7 |
| A3 | 80 | 5 | 10 |

Критерии:

| Критерий | Направление | Порядок | Уступка |
| --- | --- | ---: | ---: |
| K1 | max | 1 | 10 |
| K2 | min | 2 | 10 |
| K3 | max | 3 | не применяется |

Ожидаемый результат: `A1`.

### Пример 2

Альтернативы:

| Альтернатива | K1 | K2 | K3 |
| --- | ---: | ---: | ---: |
| A1 | 10 | 100 | 5 |
| A2 | 12 | 90 | 5 |
| A3 | 15 | 60 | 3 |

Критерии:

| Критерий | Направление | Порядок | Уступка |
| --- | --- | ---: | ---: |
| K1 | min | 1 | 3 |
| K2 | max | 2 | 15 |
| K3 | max | 3 | не применяется |

По формальному правилу уступки `90 >= 100 - 15`, поэтому на втором шаге остаются `A1` и `A2`. На последнем критерии они имеют одинаковое значение, значит ожидаемый результат программы: `A1`, `A2`.

## Протокол расчета

После нажатия кнопки "Рассчитать" приложение выводит:

- итоговую альтернативу или список равноценных альтернатив;
- итоговое множество после всех уступок;
- пошаговый протокол;
- список исключенных альтернатив по шагам.
